#### <b>一些问题</b>
##### <b>线程是不是越多越好？</b>
1. 线程的创建和销毁都是非常贵的操作
2. 线程栈会占用大量内存
3. 过多线程导致的线程上下文切换过于频繁
4. 过多线程同时唤醒导致系统锯齿状负载或瞬时负载过高（同一时间很多io准备好了）

线程数量一般按照cpu核心数量来确定

##### <b>线程池fixed模式和cached模式</b>
1. fixed模式：

    线程池中线程数量固定
2. cached模式：

    线程池中线程数量随线程池负载增大而增大，当线程空闲超过一定事件后将线程关闭
